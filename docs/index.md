# Stalwart Provider

Manages Stalwart Mail Server resources via its management API.

## Resource creation order

```
stalwart_domain
     │
     ├──► stalwart_dkim              (references domain)
     │         │
     │         └──► stalwart_dns_records  (data source, reads after DKIM exists)
     │
     └──► stalwart_group             (primary_email / aliases live on this domain)
               │
               └──► stalwart_account (member_of references group names)
```

Terraform resolves this automatically from resource references. Where plain
strings are used instead of resource references (e.g. `member_of`, or group
email addresses supplied via variables), add an explicit `depends_on`.

## Example Usage

```terraform
# Option A — bearer token (recommended)
provider "stalwart" {
  endpoint = "https://mail.example.com/api"
  token    = "api_dGVycmFmb3J..."
}

# Option B — username / password
provider "stalwart" {
  endpoint = "https://mail.example.com/api"
  username = "admin"
  password = "secret"
}

# Option C — token from environment variable STALWART_TOKEN
provider "stalwart" {
  endpoint = "https://mail.example.com/api"
}
```
