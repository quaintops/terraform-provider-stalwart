# stalwart_account (Resource)

Manages an individual account in Stalwart Mail Server.

~> `password` is **write-only** and is never read back from the API. Terraform
   tracks it in state only to detect changes. Pass it via a sensitive variable
   and avoid committing it to version control.

-> `member_of` contains plain group name strings, not resource references, so
   Terraform cannot infer the dependency automatically. Always add
   `depends_on = [stalwart_group.this]` to ensure groups exist before accounts
   are created and are not destroyed while accounts still reference them.

## Example Usage

```terraform
resource "stalwart_domain" "example" {
  name = "example.com"
}

resource "stalwart_group" "info" {
  name          = "info-example"
  primary_email = "info@${stalwart_domain.example.name}"

  enabled_permissions = ["email-send", "email-receive"]

  depends_on = [stalwart_domain.example]
}

resource "stalwart_account" "alice" {
  name        = "alice"
  description = "Alice Smith"
  password    = "changeme"

  primary_email = "alice@${stalwart_domain.example.name}"
  aliases       = ["a.smith@${stalwart_domain.example.name}"]

  member_of = [stalwart_group.info.name]

  quota = 10737418240 # 10 GB; omit or set to 0 for unlimited

  roles = ["user"]

  depends_on = [stalwart_group.info]
}
```

## Schema

### Required

- `name` (String) Account login name (e.g. sam).

### Optional

- `aliases` (List of String) Additional email addresses (aliases) for this account. Each is stored after the primary in Stalwart's emails array.
- `description` (String) Display name or description for this account.
- `member_of` (List of String) Names of groups (stalwart_group.name) this user belongs to. Groups must be created before the user. This is the recommended way to assign group membership in Stalwart.
- `password` (String, Sensitive) Account password. Stalwart stores it hashed. Changing this value updates the password.
- `primary_email` (String) Primary email address for this account (e.g. sam@domain1.com). Stored as index 0 in Stalwart.
- `quota` (Number) Mailbox quota in bytes. 0 means unlimited.
- `roles` (List of String) Roles assigned to this user (e.g. ["user"]).

### Read-Only

- `id` (String) Resource identifier. Always equal to name.

## Import

Accounts can be imported using the account name:

```shell
terraform import stalwart_account.example alice
```
