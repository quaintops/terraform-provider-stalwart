# stalwart_dkim (Resource)

Generates a DKIM signing key for a domain. Changing domain, algorithm, or selector forces recreation.

~> All arguments (`domain`, `algorithm`, `selector`) are **ForceNew**. Any
   change destroys the existing key and generates a fresh one. Update your DNS
   with the new TXT record before applying if you need to avoid a DKIM gap.

## Example Usage

```terraform
resource "stalwart_domain" "example" {
  name = "example.com"
}

resource "stalwart_dkim" "example" {
  domain    = stalwart_domain.example.name
  algorithm = "Ed25519"
  selector  = "stalwart"
}

output "dns_records" {
  value = stalwart_dkim.example.dns_records
}
```

## Import

DKIM resources cannot be imported. If a key already exists for the domain,
remove it from state and let Terraform recreate it:

```shell
terraform state rm stalwart_dkim.example
```
